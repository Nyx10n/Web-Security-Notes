# Cross-Site Scripting (XSS)
<div align=justify>Cross-Site Scripting (XSS) is a web application vulnerability that allows an attacker to execute malicious JavaScript in a victim's browser. It occurs when an application includes attacker-controlled input in a web page without handling it safely, causing the browser to interpret the input as executable code instead of plain text.
<br /><br />
Depending on the application's functionality and security controls, a successful XSS attack can lead to cookie theft (if cookies are accessible), session hijacking, phishing, website defacement, or even account compromise.
</div>
